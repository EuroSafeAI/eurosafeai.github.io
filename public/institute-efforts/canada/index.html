Methodology, definitions & caveats — read before citing

Data sources (all primary)

  • Papers & metadata: OpenReview API (/notes/search) — every accepted paper at each venue, with title, abstract, keywords, primary area, and authors. Counts are accepted-only (Poster/Spotlight/Oral), verified against each venue’s own accept tags.
  • Full text (for acknowledgments & affiliations): Semantic Scholar (title→arXiv id) + arXiv PDFs, matched by exact/near-exact title plus author-surname overlap.

What counts as “AI Safety”

A hybrid, transparent rule applied uniformly across all five venues (their primary_area taxonomies are not comparable — NeurIPS has no safety area at all — so a keyword layer is the equalizer):

Broad = trustworthy ML alignment/RLHF, interpretability, evals & red-teaming, jailbreaks/misuse, hallucination/truthfulness, privacy, fairness/bias, adversarial robustness, security (poisoning/backdoor), governance/societal.

Narrow = frontier safety the subset most tied to frontier-model risk: alignment, interpretability, evals & red-teaming, jailbreaks/misuse, hallucination/truthfulness.

Precision-first keyword rules over title/abstract/keywords, context-gated to avoid false hits; auditable per-paper.

Footprint detection

Affiliation: first-page author affiliations parsed from the PDF, matched against Canadian universities/institutes with guards (e.g. “New York University”, “Northwestern”, “University of Queensland”, “CIFAR-10” excluded; the ICML/PMLR “Vancouver, Canada” proceedings line stripped). Acknowledgments: the acknowledgments section only, matched against Canadian funders/institutes (NSERC, CIFAR AI Chair, Mila, Vector, Amii, Compute Canada, FRQNT, …).

Canada CIFAR AI Chairs (Section 4)

The roster of Canada CIFAR AI Chairs is taken from CIFAR’s official directory (its WordPress REST API, “Pan-Canadian AI Strategy” program). A safety paper is credited if any author’s name matches a chair (accent- and case-normalized, first + last name). This is exhaustive across all safety papers — author names are in the conference metadata — so it is not a sample. Caveat: name-matching may miss a chair listed under a different name form, or over-count a common name shared with a non-chair; spot-checks show the matched set is dominated by correct, well-known chairs.

Known limitations

  • Keyword classification has residual false positives/negatives in both directions (~a few % per subtopic).
  • Footprint figures cover only papers with a retrievable arXiv full text, so they are a lower bound; papers not on arXiv are excluded from that measure (but not from counts/topics).
  • 2026 venues reflect acceptances known as of the build date.
